Design a modern, distinctive, premium website UI for a podcast called “EGGS! The Podcast.”

This prompt is for the FRONT-END DESIGN PHASE ONLY. Focus on information architecture, page layouts, design system, UI patterns, navigation, content structure, and polished responsive visual design. Do not prioritize backend engineering yet. Design the site so it is clearly ready for future migration into a real codebase and future backend integration, but for now concentrate on the public-facing website experience and the supporting UI patterns.

The website should feel like a serious editorial media brand, not a generic startup landing page and not a default AI-generated Tailwind site. Avoid obvious SaaS tropes, repetitive utility-style blocks, overly soft rounded cards, generic gradients, dashboard aesthetics, and anything that feels like a cloned template. The site should feel sharper, more intentional, more editorial, more media-forward, and more culturally aware.

PRIORITY
Prioritize originality of layout, hierarchy, typography, rhythm, and composition over convenience of default component styling.

BRAND
The podcast is called “EGGS! The Podcast.”
It features conversations around creativity, entrepreneurship, branding, business building, technology, culture, expertise, and the people behind interesting work.
The site should feel like an energetic but credible archive of expert conversations.

VISUAL DIRECTION
The design should feel:
- editorial
- premium
- intelligent
- media-forward
- bold but not loud
- modern but not trendy in a disposable way
- culturally aware
- more publication than startup
- more digital magazine than SaaS product

TYPOGRAPHY
Use Sáinz as the primary display/typeface direction for headlines and prominent typography. If Sáinz is unavailable, use the closest possible substitute that carries a similarly distinctive editorial personality.
Pair it with a highly readable sans-serif for body copy, UI labels, navigation, metadata, and forms.
Use typography as a major part of the brand expression. Strong hierarchy is important. Make headings feel designed, not just enlarged body text.

COLOR
Use the existing brand colors from the current eggscast.com website as the source of truth for the palette. Sample and reuse the core visual color identity from the existing site instead of inventing an unrelated palette.
Refine those colors into a sharper, more premium editorial system with better hierarchy, contrast, and consistency.
Do not make the palette feel generic tech-startup. Preserve continuity with the current brand while elevating it.

OVERALL STYLE RULES
Aim for:
- stronger typography hierarchy
- asymmetrical editorial layouts
- more dramatic and intentional use of whitespace
- larger media moments
- more composition variety across sections
- sharper alignment and spacing rhythm
- distinctive cards, buttons, and content modules
- fewer overly rounded elements
- more visual edge and tension
- layouts inspired by modern digital publications, premium podcast brands, editorial platforms, and culture/media websites

Avoid:
- generic SaaS hero sections
- repetitive section stacks that all look the same
- endless rounded cards with shadows
- default Tailwind-looking components
- soft startup gradients
- bland tech product visuals
- simplistic template cloning
- overdesigned futuristic effects
- cluttered or noisy experimental layouts that hurt readability

SITE GOALS
The site must function as:
- the branded home for the podcast
- a searchable episode archive
- a YouTube growth funnel
- a sponsor and advertiser lead-generation surface
- a guest application and guest nomination funnel
- a newsletter growth surface
- an SEO-friendly and AI-discoverable knowledge library

The design must support:

- strong, readable text content around media
- clear headings and subheadings

- internal linking between episodes, topics, and key pages
- space for schema / structured data implementation later
- transcript visibility or transcript excerpts
- visible authorship / guest / date / topic information
- clear entity relationships between show, guest, episode, and topic

Treat each episode page like an authoritative knowledge page about that conversation.
